# Snapflip Database Schema

This document outlines the normalized PostgreSQL database schema designed for Supabase, capable of scaling from 5 to 100,000+ users.

## Conventions
- **Primary Keys:** `UUID` (autogenerated)
- **Timestamps:** `created_at`, `updated_at`, `soft_delete_at` (for safe deletion recovery up to 14 days)
- **Foreign Keys:** Enforce referential integrity with appropriate cascade/set null rules.

---

## 1. `users`
Core table for all authenticated users (photographers/creators).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | Unique user identifier |
| `email` | VARCHAR(255) | UNIQUE, NOT NULL | User's email |
| `name` | VARCHAR(255) | | User's display name |
| `role` | VARCHAR(50) | DEFAULT 'creator' | Role (creator, admin) |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `soft_delete_at`| TIMESTAMPTZ | NULL | Soft delete timestamp |

**Indexes:**
- `idx_users_email` ON `email`

---

## 2. `albums`
The core entity representing a flipbook album.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | Internal UUID |
| `user_id` | UUID | FK -> `users(id)` | Owner of the album. ON DELETE CASCADE |
| `title` | VARCHAR(255) | NOT NULL | Album title |
| `slug` | VARCHAR(100) | UNIQUE, NOT NULL | Publicly readable QR slug |
| `status` | VARCHAR(50) | NOT NULL | State: `draft`, `published`, `shared`, `archived` |
| `visibility` | VARCHAR(50) | DEFAULT 'public' | `public` or `password_protected` |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `soft_delete_at`| TIMESTAMPTZ | NULL | Triggered on soft delete, cleared if restored. |

**Indexes:**
- `idx_albums_user_id` ON `user_id`
- `idx_albums_slug` ON `slug`
- `idx_albums_status` ON `status`

---

## 3. `album_photos`
Tracks images associated with an album (max 50 per album).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `album_id` | UUID | FK -> `albums(id)` | ON DELETE CASCADE |
| `storage_file_id`| UUID | FK -> `storage_files(id)` | Link to raw storage metadata |
| `order_index` | INT | NOT NULL | Order of the photo in the album |
| `orientation` | INT | DEFAULT 0 | 0, 90, 180, 270 |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | |

**Indexes:**
- `idx_album_photos_album_id` ON `album_id`

---

## 4. `storage_files`
Tracks physical files in the abstracted storage provider (Google Drive, Cloudinary, etc.).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `provider` | VARCHAR(50) | NOT NULL | e.g. `google_drive`, `supabase` |
| `provider_id`| VARCHAR(255) | NOT NULL | The ID inside the provider |
| `file_type` | VARCHAR(50) | NOT NULL | `original`, `optimized`, `thumbnail` |
| `url` | TEXT | NOT NULL | Resolved direct URL |
| `size_bytes` | BIGINT | | File size |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |

**Indexes:**
- `idx_storage_files_provider_id` ON `provider_id`

---

## 5. `album_passwords`
Stores bcrypt-hashed passwords for protected albums. Separated for security.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `album_id` | UUID | FK -> `albums(id)`, UNIQUE | ON DELETE CASCADE |
| `password_hash`| VARCHAR(255) | NOT NULL | Bcrypt hashed password |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | |

---

## 6. `drafts`
Temporary or auto-saved working copies of albums before they are published.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `album_id` | UUID | FK -> `albums(id)`, UNIQUE | ON DELETE CASCADE |
| `payload` | JSONB | NOT NULL | Snapshot of working state |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | |

---

## 7. `album_shares`
Tracks sharing configurations and share links.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `album_id` | UUID | FK -> `albums(id)` | ON DELETE CASCADE |
| `share_url` | TEXT | NOT NULL | The generated shareable link |
| `qr_code_url`| TEXT | | URL to generated QR code asset |
| `expires_at` | TIMESTAMPTZ | NULL | Optional expiration date |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |

---

## 8. `album_analytics`
Tracks engagement metrics for an album.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `album_id` | UUID | FK -> `albums(id)` | ON DELETE CASCADE |
| `event_type` | VARCHAR(50) | NOT NULL | `view`, `qr_open`, `download`, `share` |
| `visitor_hash` | VARCHAR(255) | NOT NULL | Anonymized unique visitor identifier |
| `device` | VARCHAR(100) | | e.g. `mobile`, `desktop` |
| `browser` | VARCHAR(100) | | e.g. `chrome`, `safari` |
| `country` | VARCHAR(100) | | Geo-located country |
| `referrer` | TEXT | | Traffic source |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |

**Indexes:**
- `idx_album_analytics_album_id` ON `album_id`
- `idx_album_analytics_event_visitor` ON (`event_type`, `visitor_hash`)

---

## 9. `activity_logs`
Audit trail for significant actions (publishing, sharing, deleting).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PRIMARY KEY | |
| `user_id` | UUID | FK -> `users(id)` | ON DELETE SET NULL |
| `action` | VARCHAR(100) | NOT NULL | e.g., `album.published`, `album.deleted` |
| `metadata` | JSONB | | Context payload (e.g. `album_id`) |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |

**Indexes:**
- `idx_activity_logs_user_id` ON `user_id`
